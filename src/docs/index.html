🚀 API REST de Tareas

Proyecto Educativo - SC-502 Ambiente WebCliente Servidor

📋 Información del Proyecto

  • Base URL: http://localhost:8080/api/
  • phpMyAdmin: http://localhost:8081/
  • Base de Datos: MySQL 8.0
  • Lenguaje: PHP 8.2
  • Contenedores: Docker & Docker Compose

📝 Descripción

Este API REST permite gestionar tareas con operaciones CRUD completas. Está diseñado para enseñar a los estudiantes:

  • Principios de APIs REST
  • Métodos HTTP (GET, POST, PUT, DELETE)
  • Contenedorización con Docker
  • Conexión entre PHP y MySQL
  • Manejo de respuestas JSON

🛠 Endpoints Disponibles

GET Obtener todas las tareas

URL: /api/

Descripción: Obtiene todas las tareas con información del usuario asignado.

Respuesta 200 OK: 
{
  "datos": [
    {
      "id_tarea": 1,
      "id_usuario": 1,
      "estado": "pendiente",
      "titulo": "Completar proyecto final",
      "descripcion": "Terminar el desarrollo del API REST",
      "nombre_usuario": "Juan Pérez",
      "email": "juan@example.com",
      "fecha_creacion": "2025-01-15 10:30:00"
    }
  ],
  "total": 1,
  "mensaje": "Todas las tareas obtenidas"
}

GET Obtener tarea por ID

URL: /api/{id}

Descripción: Obtiene una tarea específica por su ID.

Ejemplo: /api/1

GET Filtrar tareas

URL: /api/?estado={estado} o /api/?usuario={id_usuario}

Descripción: Filtra tareas por estado o usuario.

Ejemplos:

  • /api/?estado=pendiente
  • /api/?usuario=1

POST Crear nueva tarea

URL: /api/

Descripción: Crea una nueva tarea.

Cuerpo de la petición:

{
  "id_usuario": 1,
  "estado": "pendiente",
  "titulo": "Nueva tarea",
  "descripcion": "Descripción de la tarea"
}
Respuesta 201 Created: 
{
  "datos": {
    "id_tarea": 6,
    "id_usuario": 1,
    "estado": "pendiente",
    "titulo": "Nueva tarea",
    "descripcion": "Descripción de la tarea",
    "fecha_creacion": "2025-01-15 14:30:00"
  },
  "mensaje": "Tarea creada exitosamente"
}

PUT Actualizar tarea

URL: /api/{id}

Descripción: Actualiza una tarea existente.

Ejemplo: /api/1

Cuerpo de la petición:

{
  "estado": "completada",
  "titulo": "Tarea actualizada"
}

DELETE Eliminar tarea

URL: /api/{id}

Descripción: Elimina una tarea por su ID.

Ejemplo: /api/1

📊 Estados Válidos

  • pendiente - Tarea sin iniciar
  • en_proceso - Tarea en desarrollo
  • completada - Tarea finalizada
  • cancelada - Tarea cancelada

🧪 Pruebas con cURL

Puedes probar el API usando estos comandos cURL:

Obtener todas las tareas:

curl -X GET http://localhost:8080/api/

Crear una nueva tarea:

curl -X POST http://localhost:8080/api/ \
  -H "Content-Type: application/json" \
  -d '{
    "id_usuario": 1,
    "estado": "pendiente",
    "titulo": "Tarea de prueba",
    "descripcion": "Esta es una tarea de prueba"
  }'

Actualizar una tarea:

curl -X PUT http://localhost:8080/api/1 \
  -H "Content-Type: application/json" \
  -d '{
    "estado": "completada"
  }'

Eliminar una tarea:

curl -X DELETE http://localhost:8080/api/1

🚀 Cómo ejecutar el proyecto

  1. Asegúrate de tener Docker y Docker Compose instalados
  2. Clona o descarga este proyecto
  3. Abre una terminal en la carpeta del proyecto
  4. Ejecuta: docker-compose up -d
  5. Espera a que los contenedores se inicien
  6. Visita http://localhost:8080 para ver esta documentación
  7. Usa http://localhost:8081 para acceder a phpMyAdmin

📚 Para Estudiantes

Este proyecto te ayudará a entender:

  • REST: Cómo diseñar APIs siguiendo principios REST
  • HTTP: Uso correcto de métodos HTTP y códigos de estado
  • Docker: Containerización de aplicaciones web
  • PHP: Desarrollo backend con PHP y PDO
  • MySQL: Diseño de base de datos y relaciones
  • JSON: Formato de intercambio de datos

💡 Consejos para el desarrollo:

  • Usa herramientas como Postman o Insomnia para probar el API
  • Revisa los logs de Docker con docker-compose logs
  • Modifica el código en la carpeta src/ y los cambios se reflejarán automáticamente
  • Usa phpMyAdmin para visualizar y modificar los datos de la base de datos
